Introdução

O FuelPHP é um framework conduzido pela comunidade e está aberto para contribuições de qualquer um que sinta que pode melhorá-lo ou adicionar ou funcionalidade que levará o framework a um nível mais alto. Existem múltiplas formas de fazer isso. Desde de escrever tutoriais, criar e publicar bibliotecas, contribuir para as partes oficiais do Fuel (Core, Orm, Docs etc...).

Correção de Bugs

Quando você encontra um bug ou acha um na lista de issues, que você possa resolver ou já resolveu, e quer retribuir/contribuir com o framework. Use o GitHub para enviar um pull request. O pull será de acordo com a versão que você está executando. Siga os passos a seguir quando você for fazer um pull request:

  • Minha versão atual continua sendo suportada?
    Se não for, atualize para uma versão suportada e verifique se o bug continua existindo.
  • Faça o checkout da sua versão em seu repositório local
    $ git checkout -b 1.2/develop origin/1.2/develop (bugfix branch para a versão atual )
  • Corrija o bug e faça o commit no seu repositório local
    Use uma mensagem de commit descritiva e compreensível $ git commit -m "Essa é minha descrição incrível."
  • Faça o push para sua conta no GitHub
    $ git push
  • Faça o pull request
    Faça o pull request no mesmo branch que você está (ou no qual você se baseou). Use a interface "update commit range" para direcionar a sua requisição para o branch correto. Os pull requests são aceitos apenas no branch da versão atual (1.2/develop) ou no branch da próxima versão em desenvolvimento (1.3/develop).
  • Aguarde pelo seu momento de fama
    Assim que sua requisição for mesclada, você fará parte da lista da Fama dos contribuidores.

Contribuições não relacionadas a bugs

As contribuições não relacionadas a bugs são novas funcionalidades, otimizações e todas as outras alterações. Todas essas requisições tem que ser feitas no branch mais recente de desenvolvimento. Sendo assim faça o pull do branch 1.3/develop, faça as alterações e então um pull request.
